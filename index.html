Renkleri, Fontları ve Boyutları Aktar

Figma'da tasarladığın her rengi, fontu ve köşe yuvarlaklığını tek bir dosyaya yazman gerekiyor. Bu dosya bütün projenin "renk paleti" gibi çalışıyor. Bir şeyi değiştirince her yer değişiyor.

Dosyanın yolu: styles/tokens.css

Aç, içindeki değerleri Figma'daki gerçek renklerinle değiştir. Şu an hepsi #000000 yazıyor:

--color-primary: #6C8AFF; /* ana buton rengi */ --color-primary-hover: #5A78EF; /* üstüne gelince */ --color-secondary: #2A2A3A; /* ikincil renk */ --color-background: #0F0F0F; /* sayfa arka planı */ --color-surface: #1A1A1A; /* kartların rengi */ --color-text-primary: #FFFFFF; /* ana yazı rengi */ --color-text-secondary: #9A9AAA; /* soluk yazılar */ --color-destructive: #EF4444; /* hata/sil kırmızısı */ --color-success: #10B981; /* başarı yeşili */ --font-sans: 'Inter'; /* Figma'daki fontu yaz */ --radius-md: 0.5rem; /* köşe yuvarlaklığı */
⚠️ Sadece #000000 yazan satırları değiştir. Yapının geri kalanına dokunma.

.env Dosyasını Oluştur

Proje klasöründe terminali aç ve şu komutu çalıştır. Bu, örnek dosyayı gerçek dosyaya kopyalar:

cp .env.example .env.local

Sonra .env.local dosyasını aç ve aşağıdaki key'leri doldur. Her birini nereden alacağın tabloda:

DeğişkenNereden Alınır

NEXT_PUBLIC_SUPABASE_ANON_KEYSupabase → Projen → Settings → API
SUPABASE_SERVICE_ROLE_KEYSupabase → Projen → Settings → API
DATABASE_URLSupabase → Settings → Database → Connection String → Prisma

2

Veritabanını Oluştur

Bu komut Prisma şemasını okuyup Supabase'de bütün tabloları oluşturur. Tek seferlik çalıştır:

npx prisma migrate dev --name init

Tablolar oluştuktan sonra Supabase SQL Editor'ı aç ve paket verilerini ekle:

INSERT INTO packages (id, name, daily_ai_limit, price) VALUES (gen_random_uuid(), 'student', 10, 0), (gen_random_uuid(), 'clinic_pro', -1, 299), (gen_random_uuid(), 'enterprise', -1, 999);
💡 -1 sınırsız demek. Öğrenci günde 10, Pro ve Kurumsal sınırsız.
3

Güvenlik Kurallarını Yaz (RLS)

Bu kurallar sayesinde hiç kimse başkasının hastasını, vakasını veya notunu göremez. Supabase SQL Editor'da çalıştır:

-- Güvenliği aç ALTER TABLE patients ENABLE ROW LEVEL SECURITY; ALTER TABLE cases ENABLE ROW LEVEL SECURITY; ALTER TABLE notes ENABLE ROW LEVEL SECURITY; ALTER TABLE sessions ENABLE ROW LEVEL SECURITY; ALTER TABLE pomodoro_logs ENABLE ROW LEVEL SECURITY; -- "Sadece kendin görebilirsin" kuralları CREATE POLICY "users_own_patients" ON patients FOR ALL USING (user_id = auth.uid()::text); CREATE POLICY "users_own_cases" ON cases FOR ALL USING (user_id = auth.uid()::text); CREATE POLICY "users_own_notes" ON notes FOR ALL USING (user_id = auth.uid()::text); CREATE POLICY "users_own_sessions" ON sessions FOR ALL USING (user_id = auth.uid()::text); CREATE POLICY "users_own_pomodoro" ON pomodoro_logs FOR ALL USING (user_id = auth.uid()::text);
4

Kendine Admin Yetkisi Ver

Önce uygulamaya normal kullanıcı olarak kayıt ol. Sonra Supabase SQL Editor'da şunu çalıştır — email adresini değiştir:

UPDATE users SET role = 'admin' WHERE email = 'senin@emailin.com';

Artık /admin sayfasına erişebilirsin. Normal kullanıcılar giremez.

5

Middleware'i Tamamla

Şu an middleware.ts iskelet hâlinde. Gerçek auth kontrolünü ekle. Dosyanın tüm içeriğini şununla değiştir:

import { createServerClient } from '@supabase/ssr' import { NextResponse, type NextRequest } from 'next/server' export async function middleware(request: NextRequest) { const { pathname } = request.nextUrl const response = NextResponse.next() const supabase = createServerClient( process.env.NEXT_PUBLIC_SUPABASE_URL!, process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!, { cookies: { get: (n) => request.cookies.get(n)?.value, set: () => {}, remove: () => {}, } } ) const { data: { user } } = await supabase.auth.getUser() if (!user && pathname.startsWith('/dashboard')) { return NextResponse.redirect(new URL('/login', request.url)) } if (!user && pathname.startsWith('/admin')) { return NextResponse.redirect(new URL('/login', request.url)) } return response } export const config = { matcher: ['/dashboard/:path*', '/admin/:path*'] }
