## AI SİSTEMİ

Model Sabitleri (lib/ai/client.ts):
  AI_MODELS.FAST      = claude-3-5-sonnet-20241022   → AI Tanı, Vaka RPG, Terminal
  AI_MODELS.EFFICIENT = claude-3-5-haiku-20241022    → AI Asistan, Günlük Brifing

AI Chat Endpoint: POST /api/ai/chat
  Request body: { message: string, model: 'FAST'|'EFFICIENT', caseId?: string }
  Response: { response: { type: 'text', text: string } }
  Hata (limit): 429 → { error: 'AI limit exceeded' }

Limit Sistemi:
  student     → günde 10 AI isteği
  clinic_pro  → sınırsız (-1)
  enterprise  → sınırsız (-1) + öncelikli

## AUTH AKIŞI

1. Kullanıcı /login → form action={login} server action çağrılır
2. Supabase Auth ile giriş yapılır
3. Başarılıysa /dashboard'a redirect
4. middleware.ts: /dashboard ve /admin için auth kontrolü yapar
5. Auth yoksa /login'e redirect

Server Action'lar (lib/actions/auth.ts):
  login(formData)    → Promise<void> (hata: throw new Error)
  register(formData) → Promise<void>
  logout()           → Promise<void>

## ÖNEMLİ KURALLAR (Bozma, uyu)

1. Renk: Asla hardcode renk yazma. Her zaman var(--color-xxx) veya bg-[var(--color-xxx)] kullan.
2. Server/Client: Veri çeken sayfalar Server Component. Etkileşimli kısımlar 'use client'.
3. Auth: Her korumalı route app/(dashboard)/ altında olmalı — layout.tsx zaten kontrol ediyor.
4. AI: Her AI isteği önce /api/ai/chat endpoint'inden geçmeli. Direkt client'ta Anthropic çağırma.
5. Import: UI bileşenlerini @/components/ui'den, layout'ları @/components/layout'tan import et.
6. Tip: Tüm tipler @/types'tan import edilmeli. Yeni tip gerekirse types/domain.ts'e ekle.
7. Form: Server Action kullanırken return tipi Promise<void> olmalı. Hata için throw new Error().
8. Yorum: Koda yorum ekleme (sadece gerekirse).
